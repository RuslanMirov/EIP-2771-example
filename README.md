## Getting started

```bash
# 1. Install dependencies
npm install --legacy-peer-deps

# 2. Compile contracts  (uses bundled solc — no network required)
npm run compile

# 3. Run all 27 tests
npm test

# 4. Run the live demo
npm run demo

# Or compile + test in one shot
npm run build
```

> **Note on `--legacy-peer-deps`**: hardhat v2 plugins ship with the `hh2`
> tag and have overlapping peer-dep declarations.  The flag is required once
> at install time; everything works normally afterwards.

## Architecture

```
Owner (key A)                      Relayer (key B)
    │                                     │
    │  1. Build ForwardRequest            │
    │     Sign with EIP-712 (0 gas)       │
    │                                     │
    │ ────── { request, signature } ─────►│
    │                                     │
    │                                     │  2. forwarder.execute(request, sig)
    │                                     │     relayer pays ALL gas
    │                                     ▼
    │                             MinimalForwarder
    │                             ├─ verify EIP-712 signature
    │                             ├─ check & increment nonce
    │                             └─ target.call( data ++ owner_address )
    │                                            └──────────────────────┘
    │                                              EIP-2771 calldata trick
    │                                     │
    │                                     ▼
    │                              SecureVault
    │                              _msgSender() reads last 20 bytes
    │                              of calldata  →  resolves to owner ✓
    │                              onlyOwner modifier passes ✓
    │                              state updated ✓
```

## Contracts

### `MinimalForwarder.sol`

| Item | Detail |
|---|---|
| `DOMAIN_SEPARATOR` | Immutable EIP-712 domain (name, version, chainId, address) |
| `getNonce(address)` | Current nonce — include in ForwardRequest before signing |
| `verify(req, sig)` | Pure off-chain pre-check — no gas, no state change |
| `execute(req, sig)` | Relay entry point — anyone can call, relayer pays gas |

`ForwardRequest` struct:
```solidity
struct ForwardRequest {
    address from;    // original signer (owner)
    address to;      // target contract
    uint256 value;   // ETH to forward
    uint256 gas;     // gas limit for inner call
    uint256 nonce;   // replay protection
    bytes   data;    // ABI-encoded calldata
}
```

### `SecureVault.sol`

| Function | Guard | Description |
|---|---|---|
| `setCounter(uint256)` | `onlyOwner` | Update counter, emit `CounterSet` |
| `setLabel(string)` | `onlyOwner` | Update label string, emit `LabelSet` |
| `transferOwnership(address)` | `onlyOwner` | Emit `OwnershipTransferred` |
| `deposit()` | `onlyOwner` | Receive ETH via meta-tx |
| `withdraw(address, uint256)` | `onlyOwner` | Send ETH |
| `getState()` | view | owner, counter, label, totalUpdates, balance |

Custom errors: `NotOwner(address caller)` · `ZeroAddress` · `InsufficientBalance`

## Test suite — 27 passing

```
1. Deployment
   ✔ forwarder stores correct domain separator
   ✔ vault owner == deployer
   ✔ vault recognises the forwarder as trusted
   ✔ vault does NOT trust arbitrary addresses

2. Direct calls (owner pays gas)
   ✔ owner can setCounter directly
   ✔ owner can setLabel directly
   ✔ non-owner setCounter reverts with NotOwner
   ✔ non-owner setLabel reverts with NotOwner

3. Meta-transactions (relayer pays gas)
   3a. setCounter via meta-tx
     ✔ relayer can relay a valid setCounter signed by owner
     ✔ emits CounterSet with owner as `by` (not relayer)
   3b. setLabel via meta-tx
     ✔ relayer can relay setLabel signed by owner
     ✔ emits LabelSet with owner as `by`
   3c. Sequential meta-txs
     ✔ nonce increments correctly across multiple relays

4. Replay protection
   ✔ replaying the same signature reverts
   ✔ verify returns false for a stale (already-used) nonce

5. Signature forgery
   ✔ attacker signing with own key but claiming from=owner is rejected
   ✔ calling vault directly as relayer is rejected by onlyOwner
   ✔ sig with wrong nonce is rejected even if ECDSA is valid
   ✔ truncated signature reverts

6. Ownership transfer
   ✔ owner can transfer ownership directly
   ✔ new owner can mutate state after transfer
   ✔ old owner is locked out after transfer
   ✔ transferOwnership to zero address reverts
   ✔ owner can transfer ownership via meta-tx (relayer pays gas)

7. Gas benchmarks
   ✔ setCounter direct  < 100 000 gas  (actual: 69 916)
   ✔ setCounter meta-tx < 200 000 gas  (actual: 104 933)

8. getState()
   ✔ returns consistent state after multiple meta-txs
```

## Security properties

| Threat | Defence |
|---|---|
| Replay attack | Per-signer nonce consumed atomically on `execute()` |
| Signature forgery | `ecrecover(EIP-712 hash)` must equal `req.from` |
| Cross-chain replay | `chainId` baked into immutable `DOMAIN_SEPARATOR` |
| Untrusted forwarder | Only the address set at deploy time appends `_msgSender()` suffix |
| Direct attacker call | `_msgSender()` falls back to `msg.sender` → `onlyOwner` rejects |
| Griefing (wrong ETH) | `require(msg.value == req.value)` in `execute()` |
